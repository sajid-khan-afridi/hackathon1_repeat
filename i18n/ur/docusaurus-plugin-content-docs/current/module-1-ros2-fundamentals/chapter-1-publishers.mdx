# باب 1: ROS 2 Publishers اور Subscribers

اس باب میں، ہم <GlossaryTooltip termId="ros-2">ROS 2</GlossaryTooltip> کے سب سے بنیادی مواصلاتی نمونے کو دریافت کریں گے - **publish-subscribe model**۔ یہ ROS 2 nodes کے درمیان معلومات کے تبادلے کی بنیاد ہے۔

## سیکھنے کے مقاصد

اس باب کو مکمل کرنے کے بعد، آپ:
- سمجھیں گے کہ <GlossaryTooltip termId="node">nodes</GlossaryTooltip>، <GlossaryTooltip termId="topic">topics</GlossaryTooltip>، <GlossaryTooltip termId="publisher">publishers</GlossaryTooltip>، اور <GlossaryTooltip termId="subscriber">subscribers</GlossaryTooltip> کیا ہیں
- اپنا پہلا ROS 2 publisher اور subscriber بنائیں گے
- message types اور QoS settings کے بارے میں سیکھیں گے
- عام مواصلاتی مسائل کو حل کرنا سیکھیں گے

## 1.1 Publish-Subscribe Model کو سمجھنا

### اہم تصورات

**Nodes** انفرادی پروسیسز ہیں جو computation کرتے ہیں۔ ROS 2 میں، nodes کسی بھی روبوٹکس ایپلیکیشن کے بنیادی اجزاء ہیں۔

**Topics** نامزد بسیں ہیں جن پر nodes پیغامات کا تبادلہ کرتے ہیں۔ ایک topic کے متعدد publishers اور متعدد subscribers ہو سکتے ہیں۔

**Publishers** وہ nodes ہیں جو topics پر پیغامات بھیجتے ہیں۔

**Subscribers** وہ nodes ہیں جو topics سے پیغامات وصول کرتے ہیں۔

```python
# بنیادی publisher تصور
# ┌─────────────┐    message    ┌─────────────┐
# │  Publisher  │ ──────────────► │  Subscriber │
# │    Node     │                │    Node     │
# └─────────────┘                └─────────────┘
#
#            Topic: /robot_status
```

## 1.2 اپنا پہلا Publisher بنانا

آئیے ایک سادہ publisher بناتے ہیں جو ہر سیکنڈ "Hello World" پیغام بھیجتا ہے۔

<PersonalizedSection language="python">

### <GlossaryTooltip termId="python">Python</GlossaryTooltip> Implementation

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class SimplePublisher(Node):
    def __init__(self):
        super().__init__('simple_publisher')

        # Topic نام اور queue سائز کے ساتھ publisher بنائیں
        self.publisher_ = self.create_publisher(
            String,
            'hello_topic',
            10
        )

        # وقتاً فوقتاً publishing کے لیے timer بنائیں
        timer_period = 1.0  # سیکنڈ
        self.timer = self.create_timer(
            timer_period,
            self.timer_callback
        )
        self.counter = 0

        self.get_logger().info('Simple publisher node شروع ہو گیا')

    def timer_callback(self):
        msg = String()
        msg.data = f'Hello World! شمار: {self.counter}'
        self.publisher_.publish(msg)
        self.get_logger().info(f'شائع کیا: "{msg.data}"')
        self.counter += 1

def main(args=None):
    rclpy.init(args=args)
    node = SimplePublisher()

    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```

</PersonalizedSection>

:::tip کوڈ کی وضاحت
- `rclpy.init()`: ROS 2 Python کلائنٹ لائبریری کو شروع کرتا ہے
- `create_publisher()`: ایک نیا publisher بناتا ہے
- `create_timer()`: ایک timer بناتا ہے جو callback فنکشن کو کال کرتا ہے
- `rclpy.spin()`: node کو چلتا رکھتا ہے
:::

## 1.3 اپنا پہلا Subscriber بنانا

اب ایک subscriber بناتے ہیں جو ہمارے publisher کے پیغامات وصول کرے:

```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class SimpleSubscriber(Node):
    def __init__(self):
        super().__init__('simple_subscriber')

        # Subscriber بنائیں
        self.subscription = self.create_subscription(
            String,
            'hello_topic',
            self.listener_callback,
            10
        )
        self.get_logger().info('Simple subscriber node شروع ہو گیا')

    def listener_callback(self, msg):
        self.get_logger().info(f'موصول ہوا: "{msg.data}"')

def main(args=None):
    rclpy.init(args=args)
    node = SimpleSubscriber()

    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()
```
